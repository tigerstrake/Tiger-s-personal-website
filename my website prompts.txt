⏺ That changes things. Yes, you can do password-protected downloads on Cloudflare without touching the static site.

  Two practical options:

  Option 1 — Cloudflare Workers (simplest)
  Write a small Worker that intercepts requests to /docs/*, checks for a password (via a cookie or query param), and either serves the file or returns a 401. The static site stays exactly as is. Free on the Workers free tier. About 20 lines of
  code.

  Option 2 — Cloudflare Access (Zero Trust)
  If you're on Cloudflare Pages or have Access set up, you can protect specific URL paths with authentication — email OTP, Google login, etc. No code required, just a policy in the dashboard. Free for up to 50 users.

  For your use case (probably protecting a résumé PDF or schematics from casual indexing), the Worker approach is the right call — it's lightweight and gives you a simple password prompt. You'd deploy it as a Worker route over your domain that
  only matches /docs/* or whatever path you want to protect.

  Want me to write the Worker? I'd need to know:
  1. Which files/paths you want to protect (e.g. all of /docs/, specific files)
  2. Whether a single shared password is fine, or you want something more granular
